## 颜色单位
- CSS可以直接使用颜色的单词来表示不同的颜色
    + red
    + blue
    + green
- 也可以使用RGB值来表示不同颜色
    + rgb(红色的浓度，绿色浓度，蓝色浓度)
    + 颜色浓度使用 0-255 或者 0%-100% 表示
- 也可以使用十六进制表示
    + #ff0000
    + 前两个表示红色浓度
    + 中间绿色
    + 最后两个蓝色、
    + 如果是这种形式 #aabbcc 可以简写为 #abc
+ HSL 值
    + H 色相 (0 - 360)
    + s 饱和度 (颜色浓度 0%-100%)
    + L 亮度 (颜色亮度 0%-100%)
+ HSLA
    + a 是透明度

Web 安全色 216种
00 33 66 99 cc ee ff 组合而成
6 * 6 * 6 = 216


## 像素
+ 在前端开发中像素主要分为两种情况讨论: css像素 和 物理像素
+ 物理像素- 屏幕中的像素点
+ css像素: 编写网页时,使用的像素
    + 浏览器在显示网页时,需要将css像素转换为物理像素
    + 一个css像素最终由几个物理像素显示由浏览器决定
        + 默认情况下在pc端,一个css像素 == 一个物理像素

## 视口(viewport) 
+ 视口就是屏幕中用来显示网页的区域
+ 可以通过查看视口的大小,来观察css 像素和物理像素的比值 (观察 html标签的长度)
+ 默认情况下
    + 视口宽度 1920(css像素) = 1920(物理像素)
        + 此时 css像素:物理像素 = 1:1
+放大两倍
    + 视口宽度 960(css像素) 1920(物理像素)
        + 此时 css像素:物理像素 = 1:2
    + 所以我们可以通过改变视口的宽度来改变 css像素和物理像素的比值

## 注意
+ 在不同的屏幕中,单位像素的大小是不同的,像素越小越清晰
+ 智能手机像素点 远远小于 计算机的像素点
+ 默认情况下,移动端的网页都会讲视口设置为 980px(css像素)
    + 如果网页过大,会自动缩小以显示整个网页
    + 大部分pc端网站都可以用在移动端中正常浏览,但是往往不会有一个好的体验
    + 为了解决这个问题,需要为移动端专门设计网站  

+ 编写移动端页面时,必须要确保有一个比较合理的像素比:
    + 可以通过meta标签来设置视口大小
    + 将网页的视口设置为完美的视口
        + meta name="viewport" content="width=device-width, initial-scale=1.0"
        + width= device-width 就是将视口宽度设置为设备屏幕宽度
        + initial-scale 是基础缩放
        + 上面两个效果一样,一起写是防止意外,同时在旋转的时候不变
