# 单文件组件 SFC

## 为什么要使用 SFC

1. **使用熟悉 HTML, JS, CSS 语法编写模板**
2. **让本来就强相关的内容(HTML, CSS, JS) 内聚 d**
3. **组件作用域内的 css**

我觉得主要就这几点..

在轻量级的场景下(例如: 只需要给静态 HTML 添加一些简单的交互), 可以试试`petite-vue`, 它是一个 6 kB 左右、预优化过的 Vue 子集，更适合渐进式增强的需求。

## SFC 如何工作的

SFC 文件需要由`@vue/compiler-sfc`编译为标准的 javascript 和 css, 一个编译后的 SFC 是一个标准的 javaScript(ES)模块.这也意味着在构建配置正确的前提下，你可以像导入其他 ES 模块一样导入 SFC

## 如何看待关注分离点

传统前端将 HTML, CSS, JavaScript 分离为三大块(根据文件功能划分), 然后通过 link, script 交织在一起(虽然 Vue, React 项目打包后也是如此, 不如说前端就是如此)

但是在开发时, 我们可以将有强关联, 强依赖的内容放在一个文件中, 实际上使组件更有内聚性和可维护性。(根据关联划分)
